## 世界觀與核心玩法

- 主角：哥布林。敵人：人族
- 背景：哥布林開採深山資源過度，人族憤怒/仇恨/貪婪，想佔領哥布林裝飾好的家園
- 畫面：2D 平面，類似 Terraria
- 機制：1–4 人合作塔防，把資源裝飾自己的家、圍住核心建築

## 遊戲節奏

```
白天：1 分鐘（收集資源階段）
晚上：1 分鐘（怪物入侵階段）
開局：30 秒準備時間，提供基礎資源供玩家開始 Build
每個晝夜循環結束（繼續挑戰時）：額外 30 秒 Build Plan 準備時間，可聊天
```

### 第 0 關初始資源包

第 0 關初始資源包用於新手教學與正式開局的第一次 Build。

```md
土 x5
沙 x5
石 x5
鐵 x1
金 x1
鑽 x1
```

規則：

```md
房間共享一包。
單人 / 多人都是同一包，不依玩家數放大。
直接放入塔內共享資源欄。
只在新開局給一次。
讀取既有存檔不重複給。
```

用途：

- 讓新手可以立刻學會每種基礎方塊的用途。
- 讓玩家在第一個白天前就能嘗試建造、查看核心數值變化。
- 不給過多鐵 / 金 / 鑽，避免干擾第 1-10 關壓力曲線。

### 關卡編號邏輯
```
格式：[十位數]-[個位數]
0-1 = 第1關，每打完一個晝夜循環 → 通關數 +1
1-0 = 第10關 = 王關（個位數進位時 = 王關）
2-0 = 第20關 = 王關，以此類推
```

## 哥布林角色數值

| 數值 | 基礎值 | 說明 |
|---|---|---|
| 挖掘能力 | 10 | 每秒可敲擊 5 次 |
| 疲勞值 | 60（最多儲存 120） | 恢復速度=疲勞值數值本身，上限固定120不變 |
| 靈動能力 | 50 | 靠近核心時大幅增加核心攻擊力/攻速 |
| 背負能力 | 50 | 承重能力，決定能帶多少重量的資源 |
| 修復能力 | 50 | 可修復塔的血量 |
| 移動速度 | 50 | 地圖移動速度 |

### 霸體（核心機制）
```
哥布林玩家角色永遠不會被怪物攻擊，無死亡機制
整個遊戲中，唯一會被怪物攻擊扣血的對象 = 核心
（這是常駐被動設定，不需要任何條件觸發）
```

### 疲勞值細節
```
每個白天/晚上的恢復量 = 玩家當前疲勞值數值
例：疲勞值=100 → 每個白天/晚上恢復100點
儲存上限固定為120，不會因數值提升而改變
```

### 技能點數與升級
```
每升1級 → 獲得1點技能點
技能點可分配到上述六項數值，每點 +10%，最高10級（+100%）

計算方式：
最終數值 = (基礎值 + 裝備加成 + 卡片等固定加值) × (1 + 技能等級 × 10%)

例：挖掘能力
基礎10 + 裝備50 = 60
技能點點滿10級（+100%）→ 60 × 2 = 120
```

### 靈動能力細節
```
每 100 點 → 核心攻擊力與攻速 +10%
單人模式時，額外固定 +15%
```

### 修復能力細節
```
每秒回復血量 = 數值 / 60（無條件捨去到小數點後 2 位）
修復每秒消耗 1 疲勞值

觸發條件：玩家站在核心地基（任何連接核心的泥土格）上
操作：
  桌面：按住 R
  手機：按住 🔧 虛擬按鈕
```

### 移動速度細節
```
地圖每格 = 10px
50ms 移動單位 → 玩家每秒可走 5 格
```

## 裝備系統

| 裝備類別 | 對應能力 |
|---|---|
| 鎬子 | 挖掘能力 |
| 帽子 | 靈動能力 |
| 衣服 | 背負能力 |
| 褲子 | 修復能力 |
| 鞋子 | 移動能力 |

### 款式（重要：每個類別不是單一裝備，是10種款式的組合）
```
每個裝備類別底下，各有 10 種獨立款式（A~J）
- 款式間純外觀不同，效果機制完全相同（多開10條進度線給玩家練）
- 未來可能讓特定款式擁有付費限定特效，目前不做
- 每種款式各自獨立合成、獨立升級，互不影響

取得方式：抽獎決定，抽到哪個類別的哪個款式就是那件（詳見抽獎系統章節，待討論）
```

### 合成規則
```
只能「同類別 + 同款式」的兩個裝備合成，不能跨款式
2 個 0 級 → 合成 1 個 1 級
2 個 1 級 → 合成 1 個 2 級
... 以此類推，最高合到 10 級

尚未合成的重複品會留在玩家庫存裡（例：鎬子-E 同時持有 2級x3個、5級x1個）
合成觸發方式（手動點擊 / 自動每秒合成2個 / VIP加速）⚠️ 待設計，不影響升級數值規則
合成消耗「銀幣」這個貨幣（詳見「抽獎系統」章節），實際消耗數值待定
```

### 屬性加成計算
```
數值對應：單一款式 N 級 = 數值 N（1級=1, 2級=2 ... 10級=10）

該裝備類別總加成 = A~J 共10款式，各自取「目前持有的最高等級」數值，加總起來
（同一款式即使同時持有多個不同等級的重複品，只取最高的那個去加總，不會疊加）

例1：A~J 全部都合成到1級 → 1×10 = 總共10點
例2：A1 B2 C5 D3 E9，其餘(F~J)都是0級 → 1+2+5+3+9 = 總共20點
```

## 方塊系統

### 挖掘數值（耐久 / 重量）

| 方塊 | 耐久 | 重量 | 備註 |
|---|---|---|---|
| 沙/土/石 | 50 | 5 | 10 挖掘能力，5 下出塊 |
| 鐵 | 100 | 10 | |
| 金 | 150 | 10 | |
| 琉璃 | 500 | 20 | |
| 鑽 | 1000 | 50 | |

### 方塊屬性加成（堆疊到核心）

| 方塊 | 加成 |
|---|---|
| 沙塊 | 攻擊範圍 +1 |
| 土塊 | 血量 +1 |
| 石塊 | 防禦 +0.1 |
| 鐵塊 | 塔物理攻擊 +0.1 |
| 金塊 | 塔攻速 +0.01 次 |
| 琉璃 | 魔法傷害 +0.1%（穿透防禦力） |
| 鑽塊 | 連鎖攻擊 +0.1 人 |

### 核心攻擊與防禦機制

```
核心攻擊力   = 基礎5   + (鐵塊數量 × 0.1)
核心攻速     = 基礎1次/秒 + (金塊數量 × 0.01)
核心血量     = 基礎50  + (土塊數量 × 1)
核心防禦力   = 基礎0   + (石塊數量 × 0.1)
核心攻擊範圍 = 基礎50  + (沙塊數量 × 1)
魔法傷害%   = 0%     + (琉璃數量 × 0.1%)
連鎖攻擊人數 = 0      + (鑽塊數量 × 0.1)

目標選擇（普攻）：
- 攻擊範圍內有多隻怪時，普攻鎖定「離核心最近」的目標
- （核心 2x2，距離以怪物到核心本體的距離計算）

連鎖攻擊運作方式：
- 一次攻擊「同時」命中多個敵人（非傳遞式連鎖）
- 整數部分 = 保證命中人數，小數部分 = 機率觸發多命中1人
  例：連鎖1.1人 → 保證命中1人，10%機率額外命中第2人
- 連鎖目標選擇：以普攻主目標為中心，往外取「最近的 N 隻」，同一次攻擊內不重複命中同一隻
- 若範圍內可命中對象在本次連鎖已全部被選過一輪，連鎖數仍有剩餘時，重啟新一輪循環、可再次重複命中（避免怪數少於連鎖數時浪費連鎖）

傷害計算公式：
總傷害 = (核心攻擊力 × (1 - 敵人防禦減傷%)) + (核心攻擊力 × 魔法傷害%)
　→ 物理傷害部分受敵人防禦影響，魔法傷害部分無視防禦力，兩者同一次攻擊一起計算
```

### 核心血量與攻擊範圍機制

```
- 方塊是永久生效的消耗品：放置後不會被損壞，無數量上限
- 唯一會被怪物攻擊扣除的數值 = 核心血量
- 核心血量歸零 → 遊戲結束
- 核心攻擊覆蓋 = 所有連接核心的泥土格各自產生覆蓋後合併（非單一圓形）
  → 玩家往外擴張地基，攻擊覆蓋跟著擴大；若怪物必須沿著連通地基接近核心，建築長度會變成接觸前預射距離
- 怪物攻擊高度 = 怪物身高 + 攻擊距離
  → 基礎人類（平民）身高 2 格、攻擊距離 1 格 → 可攻擊到最高第 3 格
  → 特殊單位（跑者/猛男/盾兵/工兵）身高 3 格、攻擊距離 1 格 → 可攻擊到最高第 4 格
  → Boss 身高 4 格；Boss10 攻擊距離 1 格，Boss20 起攻擊距離 2 格
  （防止玩家純靠高塔讓近戰完全無效）
- 數值成長為線性、無上限，平衡靠怪物強度隨晚數提升來達成（細節待設計）
```

### 建築維度（重要：本遊戲是三維度，不是平面 2D）
```
本遊戲有三個維度：
  X（左右）：第一維度，水平方向
  Y（上下）：第二維度，高度方向（「可無限往上堆疊、高度不限」指的是這軸）
  Z（內外/深度）：第三維度，建築由內（背景）往外（玩家面向）蓋

「第一層 / 第二層」是 Z 軸（深度），不是 Y 軸的上下：
  - 第一層 = 背景層：泥土只能放在「背景版」上，這是地基。
  - 第二層 = 前景層：某格背景先放了泥土後，才能在它「前方」（朝玩家方向）
    蓋第二層方塊（沙/石/鐵/金/琉璃/鑽 + 梯子）。
  - 第二層方塊會「視覺上覆蓋」掉它背後那格背景泥土。

所以：
  - 「第二層必須有第一層泥土」= 該格的背景必須先是泥土（第二層放在泥土前方），
    不是「正下方（Y 軸下方）要有泥土」。
  - 背景泥土本身是一面 X-Y 平面的牆，可往上 / 往側邊延伸（= 高層地基 / 延伸地基）。
  - 連通性（找核心地基）= 在「背景泥土平面」內判定（左右 X + 上下 Y 都算相鄰），
    不穿透深度；第二層（前景）方塊不參與連通性，只要背後有連通泥土就能放。

核心本體與放置原則：
  - 核心是 2x2x2 的立方體（正面 2x2 面對玩家），水平置中、貼地坐落地面（不浮空）。
  - 核心貼著蓋原則：一格能放泥土 = 它相鄰的是核心，或相鄰泥土能一路追溯回核心（源頭是核心）。
    例：核心在 (x0~x1, y0~y1)，(x2,y1) 緊鄰核心可放泥土，即使 (x2,y0) 是空的也合法。
    → 泥土不靠正下方支撐，靠的是「連回核心」；但核心本體本身是貼地的。
  - 核心自己佔的格位不能放泥土；核心正前方的 Z（前景）也不能蓋第二層方塊，
    第二層只能蓋在核心周圍連出去的泥土前方。
```

### 建塔規則
```
核心地基定義：
- 只有與核心「連續相連」的泥土格，才算核心地基（連通性在背景泥土平面內判定）
- 孤立泥土（未連接核心）不生效，不提供攻擊覆蓋，也不算 Hitbox
- 放置與拆除都必須維持連通性：
  → 若拆除某格泥土會讓其他泥土與核心斷開，禁止拆除
  → 不能放置孤立的泥土段（放置後仍未與核心連通，則禁止放置）

核心攻擊覆蓋：
- 每個連接核心的泥土格，都會產生攻擊覆蓋
- 所有泥土格的覆蓋區合併後 = 核心可攻擊區（不是只從核心本體算一個圓形）
- 泥土蓋越遠，打得越早；如果怪物必須沿連通地基接近核心，怪物會先承受建築長度提供的預射距離

核心 Hitbox：
- 所有連接核心的泥土格，都算核心 Hitbox
- 怪物只要攻擊距離內碰到任何一段核心地基，就會造成核心扣血
- 核心仍然只有一條血（不是每段泥土各自有血）

第二層方塊（沙/石/鐵/金/琉璃/鑽，即「泥土以外」的方塊）：
- 第一層地基只有泥土（土塊）；沙、石與鐵/金/琉璃/鑽都是第二層物質
- 蓋在連通泥土的「前方」（Z 軸前景），會覆蓋背後那格背景泥土
- 背景泥土牆可往上延伸多高，前景第二層就能跟著蓋多高（高度不限指 Y 軸）
- 主要用途：資源加成、建築形狀、玩家站立/移動
- 不是防禦牆：怪物攻擊到泥土或前方第二層方塊，本質上都是核心掉血
- 不能作為厚防線阻擋核心傷害
```

### 梯子與多層建築
```
玩家擁有無限數量的「梯子」道具，與其他第二層方塊共用同一層
放置梯子 → 玩家可以往上爬
搭配第二層方塊持續堆疊 → 可蓋出二樓、三樓等多層結構
```

### 建築策略整理

| 策略 | 玩法 | 優點 | 代價 |
|---|---|---|---|
| 集中地基 | 泥土集中在核心附近 | 不容易太早被怪打到 | 攻擊覆蓋較晚 |
| 延伸地基 | 泥土往外拉線/分支 | 提早攻擊怪物，增加怪物接觸前預射距離 | 需要更多泥土，路徑與生成規則需模擬平衡 |
| 高層地基 | 泥土往上或斜向延伸 | 可能避開部分低身高近戰壓力 | 建造/移動複雜，被高身高或遠程怪克制 |

⚠️ 待確認：方塊種類目前是「沙土石鐵金琉璃鑽」，未來會擴充，新方塊的屬性加成規則待設計時再補。

## 怪物系統

### 生成規則
```
怪物從天上掉落，落點 = 目前核心攻擊範圍 + 10~20格外（隨機）
落地後無腦走直線朝核心方向移動，不會繞路、不會閃避
```
⚠️ 待確認：怪物移動速度尚未正式定案，目前模擬暫用「跟玩家基礎移動速度一樣(5格/秒)」當預設值

### 出怪時間分布
```
一波怪物（N隻）在晚上開始後的 0~5 秒內陸續分批掉落：
base = N ÷ 5（取整數）
remainder = N mod 5
前 remainder 秒，每秒多掉 1 隻；其餘秒數每秒掉 base 隻

例：N=6 → 第1秒2隻、第2~5秒各1隻
```

### 攻擊機制
```
怪物只有走到核心的Hitbox（泥土覆蓋範圍，或第二層方塊）才會開始攻擊
攻擊間隔固定 2秒1次
怪物攻擊距離預設為 1 格
基礎人類（平民）身高 2 格，特殊單位身高 3 格
Boss 身高 4 格；Boss10 攻擊距離 1 格，Boss20 起攻擊距離 2 格
```

### 怪物組成公式（隨關卡數增長）
```
平民數量(level) = 5 + ⌊level ÷ 2⌋
猛男數量(level) = ⌊level ÷ 5⌋

例：Lv1=5平民0猛男 / Lv2=6平民0猛男 / Lv3=6平民0猛男 / Lv4=7平民0猛男 / Lv5=7平民1猛男
```

詳細波次、多人倍率、Boss20 門檻、21-30 阻擋區集中記錄於 `Docs/waveplan.md`。

```
單人 / 2 人裸裝：第 10 關前必須輸
3 / 4 人裸裝：第 15 關前必須輸
50 抽：應能摸到第 10-15 關區間，但不穩定越過第 20 關
100 抽 + 第 10 關卡片：應能挑戰第 15-20 關，但不應穩定越過第 20 關
第 20 關：MVP 主要硬門檻
第 21-30 關：暫作阻擋區，不做公平通關承諾
```

MVP 定案節奏：

```
1-20 關是主要可玩與校準區。
第 10 關是第一個 Boss / 卡片入口。
第 20 關是裝備、卡片與操作共同挑戰的門檻。
21-30 關暫時用隨機血量 / 傷害增壓阻擋玩家穩定通過。
```

第 19 關單人基準波次：

```
平民 x8 + 猛男 x5 + 跑者 x4 + 盾兵 x3 + 工兵 x1
```

### 多人波次倍率（Must Solve 4）

詳細波次規則集中記錄於 `Docs/waveplan.md`。

初版多人規則：

```
N 人房普通怪數量 = 單人基準數量 x N
N 人房王關 = 出 N 隻 Boss
全部 Boss 擊殺後，才進入卡片選擇
```

Boss 不吃普通怪數量倍率，也不吃 4 人普通怪血量 +20% Debuff；Boss20 有獨立多人門檻，詳見 `Docs/waveplan.md`。

設計理由：

- 多人會同時增加挖礦、建塔、修復與靈動加成。
- 多人塔成長可能出現 1+1>2 的效果。
- 怪物數量 xN 是 MVP 階段最容易理解與測試的起點。

### 建造範圍限制（Must Solve 4）

為了避免玩家把核心 Hitbox 拉到地圖邊界或礦山區，初版限制：

```
地圖寬度：160 格
左右兩側各保留：礦山 30 格 + 生怪距離 10 格 = 40 格
額外特殊保留：10 格
可建造水平範圍：160 - 40*2 - 10 = 70 格
核心中心往左最多 35 格，往右最多 35 格
高度最多 100 格
```

若怪物生成距離不足：

```
正常生成：Hitbox 外 10-20 格
空間不足：可壓縮為 Hitbox 外 0-10 格
```

怪物仍不能生成在礦山範圍內，也不能生成在地圖外。

### 加時賽（狂暴模式）
```
晚上60秒結束時，如果怪物還沒被清光 → 進入30秒加時賽
加時賽期間，怪物攻擊力隨時間倍增：每5秒翻一倍
（5秒=2倍、10秒=4倍、15秒=8倍、20秒=16倍、25秒=32倍、30秒=64倍）

加時賽30秒結束時仍未清完怪 → 強制 GameOver（核心判定陣亡，直接進入結算）
```

### Endless Mode
```
關卡數無上限，玩家可以無限制玩下去，只要打得過就能一直繼續
30關之後，怪物數值可爆炸式提升（具體成長曲線細節尚未設計，留待後續）
```

## 礦山系統

```
地圖固定 1600×1000px（10px/格 → 160×100 格），核心固定在地圖正中央，不可移動
左右各有一座礦山（A在左、B在右），白天玩家在這裡挖礦獲得建塔資源

每座礦山可見範圍：橫向10格 × 縱向3層 = 30格
玩家挖掉底層方塊後，上方方塊會往下掉補位
新補進來的方塊用下方固定機率表隨機產生（每次進遊戲 Loading 時，預先生成一批新的隨機 Queue）

```

## 王關與卡片系統

```
每 10 個晚上 → 出現王
王被擊敗 → 出現 3 張卡片供投票

投票規則：
- 房主與玩家可投票，倒數 10 秒，不需確認，中途可換票，可不投票
- 房主強制需投票，未投則強制選第一個選項
- 房員可放棄投票（不點，或點了再取消）
- 房主票數 = 1.5 票
```

### 卡片類型
| 類型 | 效果 |
|---|---|
| 資源型 | 立刻獲得 N 個資源（沙/土/石等） |
| 能力型 | 給玩家屬性加成，本局生效 |
| 增強型 | 給塔屬性加成，本局生效 |

### 卡片價值點數

卡片對玩家顯示名稱與效果，不顯示內部價值點。

內部平衡使用：

```
第 10 關卡片：100 價值點
第 20 關卡片：150 價值點
第 30 關卡片：200 價值點
```

初版 100 價值點約等於：

| 能力軸 | 100 價值點約等於 |
|---|---:|
| 挖掘量數 | 2000 |
| 核心攻擊 | +2 |
| 核心攻速 | +0.20 |
| 核心血量 | 上限 +25，並回復 25 |
| 玩家能力 | 主要能力 +20 到 +25 |

第 20 / 30 關可以用 1.5 倍 / 2 倍價值做同模板高階卡，但攻速、靈動、連鎖這類乘法能力需要先模擬，不能無腦線性放大。

第 10 關已定基準卡：

| 卡片 | 類型 | 效果 |
|---|---|---|
| 鐵牙核心 | 增強型 | 核心攻擊 +2 |
| 金輪核心 | 增強型 | 核心攻速 +0.20 |
| 厚土外殼 | 增強型 | 核心血量上限 +25，並回復 25 |

## 抽獎系統

```
每完成 1 關（晝夜循環）獲得 1 張票券
消耗：1 張票券 = 1 次抽獎
```

⚠️ 票券發放率「1張/關」是用資源經濟錨點反推出來的數值，詳見下方「資源經濟設計錨點」章節

### 盤面機制
```
畫面呈現 8×8 視覺格子，玩家點選任一格觸發抽獎
資料結構只需要維護「剩餘各類型數量」清單，不用預先洗牌固定每一格藏什麼獎品
抽取為「不放回」：每抽中一個類型，該類型剩餘數量 -1

4種「大獎」類型（多銀幣／多金幣／中票卷／4級裝備）合計 7 格，
這 7 格全部清空後，才會生成新的一盤（重新填滿同樣比例的64格）

未來功能：自動抽獎（先不做）
```

### 新手抽獎教學

新手完成強制教學第 1 關後，系統獎勵 5 抽並強制帶玩家進入抽獎教學。

教學 5 抽使用固定結果，不使用正式隨機：

```md
3 件相同款式 0 級裝備
1 個銀幣
1 件 4 級裝備（大獎）
```

教學目的：

- 介紹抽獎盤如何抽取。
- 介紹掉落與機率入口。
- 介紹 0 級裝備可以合成 1 級。
- 介紹銀幣是裝備合成用貨幣。
- 介紹抽到 4 級裝備這類大獎時會發生什麼。

```
- 50類（5大類別 × 10款式 A~J）各 2% 機率，每次抽取都重新計算（不互斥）

```

## 資源經濟設計錨點

> 用「Top-down」方式設計：先定義玩家體感目標（要花多久時間達成），再反推數值，而不是憑感覺逐項填數字。
> 票券發放率「1張/關」就是用下面這組錨點，透過模擬抽獎機制反推出來的結果。

### 已定義的錨點
```
遊戲節奏：2 分鐘 / 關（晝夜循環）

極限玩家（設計目標）：每天遊玩 20 小時以上，2 個月內可將全部50件裝備（5類別×10款式）合成到10級
  → 20小時 ÷ 2分鐘 = 每天600關，2個月(60天) = 總共36,000關

正常玩家（參考情境）：每天遊玩 4 小時
  → 4小時 ÷ 2分鐘 = 每天120關
```

### 反推結果（模擬抽獎機制得出）
```
把全部50件裝備都合成到10級，平均需要約 34,740 次抽獎
→ 36,000關 ÷ 34,740次 ≈ 0.965，取整數 = 每關發放 1 張票券
→ 用這個比例，極限玩家平均約 58天（接近2個月）可全部裝備練滿10級，符合設計目標
```

### 正常玩家（4小時/天）練單一裝備款式的時間參考

| 目標等級 | 平均所需抽獎次數 | 換算天數（不含雙倍獎勵充能加成） |
|---|---|---|
| 3級 | 329 | 約 2.7 天 |
| 5級 | 1,001 | 約 8.3 天 |
| 7級 | 3,754 | 約 31 天（1個月） |
| 9級 | 14,630 | 約 122 天（4個月） |
| 10級 | 29,149 | 約 243 天（約8個月） |

⚠️ 上表為基準速度，實際會因「雙倍獎勵充能系統」（見下方章節）而加快，視玩家使用頻率，正常玩家每日實際收益可能再提升 25%~75%。

### 之後沿用同一套邏輯的項目
```
銀幣/金幣的「少中多」實際數值、每日商店的金幣/銀幣消耗、銀幣合成消耗，
建議都比照這個 Top-down 反推法（先定義玩家體感目標，再用模擬/計算反推數值），
而不是逐項憑感覺填數字
```

## 每日商店系統（定案）

### 規則
```
每日重置（GMT+8 00:00），6 個槽位從品項池機率抽取，等級可見、類型隨機
廣告刷新：每日最多 3 次（第 1 次免廣告；後 2 次各需看一段廣告）
每次刷新：6 個槽位全部替換
只賣帳號層道具，不賣任何場次道具
鑽石不在商店販售（付費頂層貨幣走其他管道）
```

### 品項池（8種，機率加權，6槽各自獨立抽）
| 品項 | 內容 | 售價 | 公平值 | 折扣 | 出現權重 |
|---|---|---|---|---|---|
| 金幣包小 | 200 金幣 | 5,600 銀幣 | 200金 | 70% | **20%** |
| 票券包小 | 5 張票券 | 4,200 銀幣 | 150金 | 70% | **20%** |
| 銀幣包大 | 25,000 銀幣 | 440 金幣 | 625金 | 70% | **3%** |
| 票券包中 | 20 張票券 | 420 金幣 | 600金 | 70% | **10%** |
| 0級裝備（隨機款式）| 0級×1件 | 70 金幣 | 107金 | 66% | **22%** |
| 1級裝備（隨機款式）| 1級×1件 | 170 金幣 | 240金 | 71% | **15%** |
| 2級裝備（隨機款式）| 2級×1件 | 270 金幣 | 384金 | 70% | **7%** |
| 3級裝備（隨機款式）| 3級×1件 | 450 金幣 | 640金 | 70% | **3%** |

> 4級裝備不進商店，僅抽獎盤大獎可得（保持稀缺感）

### 售價原則
```
所有商店品項皆為公平值打 66~71% 折扣，買到必賺
轉化率：1 金幣 = 40 銀幣，1 票券 = 30 金幣
玩家自行判斷要不要買，但數學上一定合算
```

## 雙倍獎勵充能系統

> 設計目的：取代原本考慮的「體力系統」。原因：體力系統若以「開房消耗」設計，無法真正限制住極限玩家的進度速度（只要不結束遊戲就不會再扣），且多人房如果是「每人各自扣自己的體力才能進房」，會跟好友揪團合作的核心體驗產生摩擦。改用「雙倍獎勵充能」處理鑽石消耗管道與正常玩家追趕速度的問題，不刻意限制極限玩家。

### 效果
```
遊戲結束（這一場從開房/繼續存檔，到核心陣亡或房主選擇存檔退出為止）的結算畫面，
玩家可以選擇消耗 1 點充能，把「這一場」不限關卡數累積的票券/金幣/銀幣全部翻倍（無上限）

設計上是給「極限玩家」用的工具：因為是結算後（已經知道這場打多深）才決定要不要用，
玩家會自然把充能留給自己表現最好的那場，跳過被隊友拖累、提早結束的場次
```

### 充能容量與回復
| 身分 | 容量上限 | 回復速度 |
|---|---|---|
| 一般玩家（F2P） | 1 點 | 每 4 小時回復 1 點（不超過上限） |
| VIP（月卡） | 3 點 | 每 4 小時回復 1 點（不超過上限） |

### 使用條件（廣告規則）
```
一般玩家：
  - 每天第一次使用：免廣告，直接套用
  - 之後（充能回滿後）要再使用：需要先看一次廣告才能套用
  - 「今天是否已用過免廣告那次」每日 GMT+8 08:00 重置

VIP玩家：
  - 全部使用都免廣告，直接套用（月卡的核心福利之一）
```

### 多人房使用範圍
⚠️ 待確認：充能是每個玩家各自帳號獨立擁有、各自決定要不要用在自己那份獎勵上（目前認知是這樣，但尚未正式確認）
