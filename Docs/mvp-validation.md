# MVP Validation：MVP 驗證標準
> 狀態：MVP 測試規格 / Must Solve 7 完成版
> 最後更新：2026-06-22
> 用途：定義 MVP 做出來後如何判斷成功、失敗，以及失敗後優先調整哪個系統。

---

## 核心問題

MVP 驗證要回答：

```md
遊戲做出來後，我們怎麼知道它是成功還是失敗？
```

不是只看玩家能不能過關，而是要判斷：

- 玩家是否理解核心循環。
- 玩家是否知道自己為什麼輸。
- 白天 / 晚上的節奏是否合理。
- 建築 Hitbox 風險是否被理解。
- 卡片是否真的改變玩法。
- 多人 Emoji 是否真的幫助合作。

---

## 測試對象

MVP 至少需要測 4 種玩家情境：

| 測試對象 | 目的 |
|---|---|
| 單人新手 | 看教學、UI、死亡原因是否清楚 |
| 單人熟手 | 看 1-10、10-15 是否有掌握空間 |
| 2 人朋友房 | 看合作、回防、Emoji 是否自然使用 |
| 4 人公開房 / 模擬公開房 | 看混亂狀態下是否仍能理解遊戲 |

---

## 測試流程

每組測試建議跑到以下節點：

| 階段 | 目標 |
|---|---|
| 第 1-3 關 | 確認玩家理解挖礦、回家、核心會被攻擊 |
| 第 4-9 關 | 確認玩家開始建塔 / 修復 / 管背包 |
| 第 10 關 | 確認第一個 Boss 與卡片入口 |
| 第 11-15 關 | 確認卡片是否帶來玩法變化 |
| 第 16-20 關 | 確認第 20 關作為硬門檻是否成立 |
| 第 21+ | MVP 只確認阻擋區是否有效，不追求公平通關 |

---

## 成功標準總表

| 驗證項目 | 合格標準 | 失敗時優先調整 |
|---|---|---|
| 白天 1 分鐘 | 玩家能完成至少 1 次有效挖礦、回家、建造或存放 | 白天時間、礦山距離、移速、背包重量 |
| 晚上 1 分鐘 | 玩家有修復、建塔、回防、短偷挖等至少 1 個有效決策 | 夜晚壓力、修復收益、核心危險提示 |
| 第 1-10 關成長 | 玩家能感覺塔、資源、裝備或操作讓自己變強 | 早期怪物、資源量、教學提示 |
| 死亡理解 | 玩家能說出自己大概為何輸 | 死亡回顧、核心警告、UI 提示、教學 |
| 建築 Hitbox | 玩家理解連通泥土會提供攻擊覆蓋，也會成為核心受擊範圍 | 放置預覽、教學、建築提示 |
| 第 10 關卡片 | 玩家能說出卡片讓自己後面打法有何不同 | 卡片文字、卡片效果、出卡規則 |
| 第 20 關門檻 | 50 / 100 抽 + 第 10 卡片可挑戰，但不應穩定越過 | Boss20、加時、16-20 壓力、卡片數值 |
| 多人 Emoji | 多人測試中至少有人使用 Emoji 表達挖礦、建築、搬運、修復或回防 | Emoji 位置、快捷鍵、提示時機 |
| 多人合作 | 玩家能自行討論或用 Emoji 表達目前想做什麼 | Emoji UI、房員欄、核心危險提示 |

## 模擬驗證項目

以下項目可以用模擬先排除明顯不合理：

| 模擬項目 | 目前 MVP 目標 |
|---|---|
| 每分鐘挖礦量 | 裸裝每人約 11.4 塊 / 白天可作為基準 |
| 背包滿包時間 | 約 20-25 秒需要回核心一次，若太頻繁要調背包 / 重量 |
| 50 抽 / 100 抽能力 | 讀 `Docs/simulation/simulationinfo.md`，不要每次重算 |
| 1-20 關壓力 | 讀 `Docs/waveplan.md` 與 `Docs/simulation/simulation-log-2.md` |
| 第 10 卡片 | 讀 `Docs/bosscard.md`，目前 18 張第 10 卡無直接破 20 |
| 正式加時 | 60 秒未清完進 30 秒加時，每 5 秒攻擊翻倍 |
| 21-30 | 暫時阻擋區，不作公平通關測試 |

## 真人實玩觀察表

測試時建議記錄下列觀察，不只記通關關卡：

| 觀察問題 | 記錄方式 |
|---|---|
| 玩家白天是否迷路或來不及回家 | 記第幾關、發生幾次、原因 |
| 玩家晚上是否有事做 | 記玩家是否修復、建塔、偷挖、站著等 |
| 玩家是否知道核心在扣血 | 記是否有看到警告後回家 |
| 玩家是否理解背包 / 塔內資源 | 記是否把資源帶回家、是否卡背包 |
| 玩家是否理解泥土連通 | 記是否嘗試放孤立泥土或拆斷地基 |
| 玩家是否理解死亡原因 | 死亡後口頭問一句「你覺得為什麼輸？」 |
| 玩家是否使用卡片改變打法 | 第 10 關後觀察 3-5 關 |
| 多人是否使用 Emoji | 記使用次數、種類、是否有人跟隨 |
| ⚠️ 是否有效 | 記有人按 ⚠️ 後，幾秒內幾人回家 |

## 失敗歸因規則

測試失敗時，不要只說「不好玩」。

先歸因到下列系統：

| 現象 | 優先歸因 |
|---|---|
| 玩家白天永遠來不及 | 時間 / 移速 / 礦山距離 / 背包 |
| 玩家晚上站著等 | 夜晚互動不足 / 修復收益不足 / 怪物壓力不足 |
| 玩家很快死但不知道為什麼 | UI 警告 / 死亡回顧 / 教學 |
| 玩家知道原因但覺得不公平 | 數值 / 怪物 / 加時 / Boss |
| 玩家不建塔 | 建築教學 / 建築收益不明 / 放置操作太麻煩 |
| 玩家不修復 | 修復收益太低 / 疲勞成本太高 / UI 不明 |
| 玩家不選卡或亂選 | 卡片文字 / 標籤 / 投票 UI |
| 多人沒有合作 | Emoji 不顯眼 / 缺少危險提示 / 公開房教學不足 |

## MVP 通過標準

MVP 可以進入實作下一階段的最低標準：

```md
1. 新手能在 10 分鐘內理解挖礦、建塔、修復、核心扣血。
2. 熟手能說出第 10 關前至少 2 個風險選擇。
3. 玩家死亡後，多數能說出主要死亡原因。
4. 第 10 關卡片能讓玩家感覺後續打法有變。
5. 第 20 關能作為硬門檻，不被 50 / 100 抽 + 第 10 卡片穩定打穿。
6. 多人測試中，Emoji 至少被自然使用一次。
7. 測試者能指出自己想再玩一次或想調整策略。
```

如果達不到，依「失敗歸因規則」調整，不直接重做整個系統。

---

## 暫不驗證

MVP 先不驗證：

```md
長期留存。
30 關後 Endless 正式平衡。
完整 30 張以上卡池。
正式稀有度。
公開房大規模社交。
商店 / 好友 / 成就完整體驗。
真金流。
```

這些等 MVP 核心循環成立後再處理。
