# Economy Simulation Spec — 資源經濟模擬規格
> 建立：2026-06-28
> 用途：讓 Codex 跑 Node.js Monte Carlo，反推「少/中/多」銀幣/金幣/票券實際數值，
>        並輸出裝備合成銀幣成本與商店價格建議。
> 輸出位置：`Docs/simulation/economy-sim-log.md`

---

## 背景知識（Codex 必讀）

### 貨幣系統
| 貨幣 | 主要來源 | 主要用途 |
|---|---|---|
| 票券 | **每通關 1 張**（已定案） | 抽獎盤（每次1張） |
| 銀幣 | 抽獎盤掉落（非場次直接賺） | 技能點升級、裝備合成 |
| 金幣 | 場次 session counter + 抽獎盤掉落 | 每日商店消費 |

### 抽獎盤（64格，不放回抽樣）
| 獎品 | 數量 | 是否大獎 |
|---|---|---|
| 少銀幣 | 7 | — |
| 中銀幣 | 3 | — |
| 多銀幣 | 2 | ✅ |
| 少金幣 | 7 | — |
| 中金幣 | 3 | — |
| 多金幣 | 2 | ✅ |
| 少票券 | 3 | — |
| 中票券 | 1 | ✅ |
| 0級裝備 | 18 | — |
| 1級裝備 | 8 | — |
| 2級裝備 | 5 | — |
| 3級裝備 | 3 | — |
| 4級裝備 | 2 | ✅ |
| **合計** | **64** | **大獎7格** |

**重置條件**：全部 7 格大獎被抽中 → 立刻重置為新的 64 格盤面。

### 玩家節奏
| 類型 | 每天關數 | 每天票券 |
|---|---|---|
| 極限玩家 | 600 關 | 600 張 |
| 正常玩家 | 120 關 | 120 張 |

### 技能點銀幣花費曲線（已定案）
```
Lv1=10, Lv2=50, Lv3=100, Lv4起每級多18%
單屬性Lv1~10合計 = 5,700 銀幣
6屬性全滿 = 34,200 銀幣
```

### 裝備合成（已定案架構，成本數值待定）
```
50件裝備（5類別×10款式），每件可升0~10級
極限玩家設計目標：2個月（36,000關）全部練滿 → 需要約 34,740 次抽獎（已反推定案）
→ 每件裝備平均需要 34,740÷50 ≈ 695 次抽
→ 0級裝備+足夠材料 → 10級 需要多少銀幣？這是模擬要輸出的數值。
```

## 模擬任務（Codex 實作為 Node.js script）

### 檔案位置
`Docs/simulation/run-economy-sim.js`（執行：`node Docs/simulation/run-economy-sim.js`）

### 任務 1：抽獎盤基礎統計

模擬 **100,000 盤**，每盤抽到大獎7格全清為止，統計：
- 每盤平均需要幾抽才能清完（= pulls_per_board）
- 每盤各獎品平均掉落次數：少/中/多銀幣、少/中/多金幣、少/中/大票券、各等級裝備

輸出格式：
```
=== 任務1：抽獎盤基礎統計（N=100,000 盤）===
平均 pulls_per_board: XX.X
每盤平均銀幣格數：少銀幣 X.XX, 中銀幣 X.XX, 多銀幣 X.XX
每盤平均金幣格數：少金幣 X.XX, 中金幣 X.XX, 多金幣 X.XX
每盤平均票券格數：少票券 X.XX, 中票券 X.XX
每盤平均裝備格數：0級 X.XX, 1級 X.XX, 2級 X.XX, 3級 X.XX, 4級 X.XX
```

### 任務 2：試算「少/中/多」候選值 → 銀幣日收入

用任務1的平均掉落次數，針對以下候選值組合逐一計算：

**候選組合（銀幣）：**
```js
const SILVER_CANDIDATES = [
  { 少: 5,  中: 20,  多: 80  },
  { 少: 10, 中: 40,  多: 150 },
  { 少: 15, 中: 60,  多: 200 },
  { 少: 20, 中: 80,  多: 300 },
];
```

**候選組合（金幣）：**
```js
const GOLD_CANDIDATES = [
  { 少: 10,  中: 40,  多: 150 },
  { 少: 20,  中: 80,  多: 300 },
  { 少: 30,  中: 120, 多: 500 },
];
```

**候選組合（票券）：**
```js
const TICKET_CANDIDATES = [
  { 少: 2, 中: 8  },
  { 少: 3, 中: 12 },
  { 少: 5, 中: 20 },
];
```

對每個銀幣×金幣×票券候選組合輸出：
- 極限玩家每日銀幣收入（600關→600票→N盤→銀幣總量）
- 正常玩家每日銀幣收入（120關）
- 極限玩家幾天能點滿全部6屬性技能點（34,200銀幣）
- 正常玩家幾天能點滿1項屬性（5,700銀幣）

### 任務 3：裝備合成成本建議

目標限制：
```
裝備每件 0→10 級所需總銀幣 = equipment_silver_cost
合理範圍：極限玩家的銀幣收入（2個月）能同時支付：
  (A) 6屬性技能點全滿：34,200 銀幣
  (B) 50件裝備全部10級：50 × equipment_silver_cost 銀幣
兩項加總不超過極限玩家2個月總銀幣收入的 90%（留10%餘裕）
```

輸出：
- 每件裝備 0→10 級建議銀幣成本
- 驗算：極限玩家2個月（36,000關）銀幣總收入 vs 技能+裝備總支出

### 任務 4：金幣日收入 & 商店價格建議

已知商店賣的全是帳號層道具（銀幣包/金幣包/票券包/裝備/鑽石），只收金幣。

計算金幣日收入（抽獎盤來源），再加上場次 session 金幣。  
**場次金幣來源尚未定案，先跑兩種假設：**
- 假設A：每關額外 +5 金幣（session counter）
- 假設B：每關額外 +10 金幣

對每個候選組合輸出：
- 正常玩家每日金幣總收入（抽獎盤 + 場次假設A/B）
- 商店每日預計消費能力（建議：讓玩家每天能買 1~2 件不嫌貴）
- 各道具建議定價區間：
  - 銀幣包小/中/大（金幣價）
  - 票券包小/中（金幣價）
  - 裝備1件（金幣價）
  - 鑽石小量（金幣價）

## 驗收標準

Codex 完成後，把 `economy-sim-log.md` 的「建議採用值」區塊貼回給開發者確認。
數字不需要完美，只需要符合以下範圍：

| 指標 | 合理範圍 |
|---|---|
| 極限玩家技能全滿（天） | 45~75 天 |
| 正常玩家單屬性技能滿（天） | 60~150 天 |
| 正常玩家每日金幣收入 | 能買1~2件商店道具 |
| 抽獎盤 pulls_per_board | 不超過 50 抽（要讓玩家感覺盤面會清空） |
