## 需要同步的核心規則

### 1. 泥土必須連接核心

新增或修正成：

```md
只有與核心連續相連的泥土，才算核心地基。
只有核心地基會提供攻擊覆蓋與修復站位。
孤立泥土不生效。
```

放置與拆除都需要維持連通性。

例：

```md
O[O]QOO
```

如果拆除 `[O]` 會讓左側泥土與核心斷開，則禁止拆除。

也禁止放置類似：

```md
OXOOO
```

因為後面的 `OOO` 沒有連到核心。

---

### 2. 泥土形狀提供攻擊覆蓋

核心攻擊範圍不是只從核心 `Q` 本體算，而是：

```md
每個連接核心的泥土格，都會產生攻擊覆蓋。
所有泥土格的覆蓋區合併後，就是核心可攻擊區。
```

因此細長泥土線是合法玩法：

```md
QOOOOOOOO
```

TLDR：

```md
泥土蓋越遠，打得越早；但也更早被怪打到。
```

---

### 3. 連接核心的泥土都是核心 Hitbox

新增或修正成：

```md
所有連接核心的泥土，都算核心 Hitbox。
怪物只要攻擊距離內碰到任何一段核心地基，就會造成核心扣血。
```

注意：這裡不是多條血量，核心仍然只有一條血。  
意思是：玩家蓋出去的泥土越多，怪物可造成核心傷害的位置也越多。

---

### 4. 怪物不是只攻擊底層

修正舊理解：

```md
怪物真正能造成傷害的位置，不限定底層。
只要怪物攻擊距離內有任一格核心 Hitbox，就能造成核心傷害。
```

建議加入：

```md
怪物可攻擊高度 = 怪物身高 + 攻擊距離
```

例：

```md
平民身高 1 格，攻擊距離 1 格 → 可攻擊到最高第 2 格
猛男身高 2 格，攻擊距離 1 格 → 可攻擊到最高第 3 格
```

這是為了避免玩家用簡單高塔讓近戰完全無效。

---

### 5. 第二層方塊不是防禦牆

請避免寫成「第二層方塊可以擋怪、被打時核心不扣血」。

目前正確方向是：

```md
怪物攻擊土層或第二層方塊，本質上都是核心掉血。
第二層方塊主要用途是資源加成、建築形狀、玩家站立/移動、美觀、多層建築。
它不是防禦牆，也不能作為厚防線阻擋核心傷害。
```

---

### 6. 修復規則

補充：

```md
玩家只要站在核心地基（土層）上，就可以修復核心。
修復需要按鍵觸發。
```

暫定操作：

```md
桌面：R
手機：🔧 虛擬按鈕
```

UI/操作細節之後再談，不需要在這次展開。

## 建築策略整理

請把 Must Solve 3 的建築策略改成以下方向：

| 策略 | 玩法 | 優點 | 代價 |
|---|---|---|---|
| 集中地基 | 泥土集中在核心附近 | 不容易太早被怪打到 | 攻擊覆蓋較晚 |
| 延伸地基 | 泥土往外拉線/分支 | 提早攻擊怪物 | Hitbox 外擴，怪也更早能打核心 |
| 高層地基 | 泥土往上或斜向延伸 | 可能避開部分近戰壓力 | 建造/移動複雜，會被高身高或遠程怪克制 |

---

## 模擬方向

Must Solve 3 後續不能用舊的「核心單一圓形射程」模擬。

後續格子化模擬應改成：

```md
1. 找出所有與核心連通的泥土格。
2. 每個泥土格各自產生攻擊覆蓋。
3. 合併所有覆蓋區，形成核心攻擊區。
4. 怪物路徑若進入攻擊區，核心可以攻擊。
5. 怪物攻擊距離內若有任一核心 Hitbox，核心扣血。
```
