# BossCard：王關卡片池與價值點數
> 狀態：MVP 定案 / Must Solve 5 已收尾
> 最後更新：2026-06-22
> 用途：集中管理第 10 / 20 / 30 關後卡片設計、價值點數、卡池與後續模擬結果。

---

## 目前問題

Must Solve 5 要解的是：卡片系統與局內流派。

這個問題卡住的是：

- 第 10 / 20 / 30 關後，卡片強度需要能按階段放大。
- 卡片如果只寫效果，之後很難判斷第 20 關給 1.5 倍、第 30 關給 2 倍時是否合理。
- 資源型卡片需要統一成「挖掘量數」或等價點數，否則不同方塊很難比較。
- 能力型、核心型、玩法型卡片也需要一個共同的平衡語言。
- 玩家看到的卡片要有玩法感，但設計文件裡要能被模擬。

---

## 卡片價值點數

卡片對玩家顯示名稱與效果，不顯示內部價值點。

內部平衡使用：

```md
第 10 關卡片：1.0 倍價值 = 100 價值點
第 20 關卡片：1.5 倍價值 = 150 價值點
第 30 關卡片：2.0 倍價值 = 200 價值點
```

第 10 關 100 價值點的初版換算：

| 能力軸 | 100 價值點約等於 | 備註 |
|---|---:|---|
| 挖掘量數 | 2000 | 資源型基準；可換成不同方塊 |
| 核心攻擊 | +2 | 鐵牙核心基準 |
| 核心攻速 | +0.20 | 金輪核心定案 |
| 核心血量 | 上限 +25，並回復 25 | 厚土外殼基準 |
| 修復能力 | +25 | 偏防守 / 拖時間，需搭配加時檢查 |
| 背負能力 | +25 | 間接提高挖礦效率與建築速度 |
| 移動速度 | +20 | 影響通勤與夜晚偷挖 |
| 靈動能力 | +20 | 影響核心攻擊與攻速，需小心乘法放大 |
| 核心範圍 | +15 | 等價需後續格子模擬驗證 |
| 連鎖能力 | +0.35 | 對多怪強，需獨立模擬 |

第 20 / 30 關放大規則：

| 關卡 | 倍率 | 例子 |
|---:|---:|---|
| 10 | 1.0x | 挖掘量數 2000、攻擊 +2、攻速 +0.20 |
| 20 | 1.5x | 挖掘量數 3000、攻擊 +3、攻速 +0.30 |
| 30 | 2.0x | 挖掘量數 4000、攻擊 +4、攻速 +0.40 |

> 注意：攻速、靈動、連鎖這類乘法或群體收益能力，第 20 / 30 關不一定能直接線性放大，後續需要模擬確認。

```md
第 10 關：金輪核心，攻速 +0.20
```

## 卡池強度分層

目前先不修偏弱卡。

設計上允許卡池存在「垃圾卡 / 功能卡 / 強卡」落差，之後可用出現權重處理。

### 分層表

| 強度層 | 卡片 | 處理 |
|---|---|---|
| 偏弱 / 功能卡 | 鐵石補強、背簍加固、輕腳步、夜修班 | 先保留，不修；可作為低權重強卡池中的普通選項 |
| 標準卡 | 老礦工手感、土倉補給、沙眼備料、老礦工手腕、縫補本能、鐵牙核心、厚土外殼 | 保留 |
| 偏強卡 | 右礦通行證、靈動鼓點、沙眼瞭望、鑽光折射、貪礦契約、高塔工法、金輪核心 | 保留，但之後注意權重 |
| 需監控 | 金輪核心、鑽光折射、沙眼瞭望、貪礦契約、高塔工法 | 後續系統改動後重測 |

### 設計備註

```md
偏弱卡不代表要刪除。
偏弱卡可以讓卡池有落差，讓玩家感覺抽到強卡更有感。
但偏弱卡不能無聊到完全不想選，因此仍需有明確用途。
```

後續可以用權重控制：

| 強度層 | 初版出現權重 |
|---|---:|
| 偏弱 / 功能卡 | 35% |
| 標準卡 | 45% |
| 偏強卡 | 20% |

> 注意：這張權重表只保留作比較用。MVP 開工時以「出卡規則」章節的固定 3 槽位為準。

## 出卡規則

### 目前問題

每次王關只給 3 張卡。

如果只用純權重抽 3 張，可能出現：

- 三張都偏弱。
- 三張都偏強。
- 三張效果很像，沒有真正選擇。
- 多人投票時看不出隊伍要選資源、輸出還是防守。

### 建議規則

MVP 先採用「固定槽位 + 小隨機」。

每次王關出 3 張：

| 槽位 | 規則 | 目的 |
|---|---|---|
| 第 1 張 | 必定標準卡 | 保底可選 |
| 第 2 張 | 55% 偏弱 / 45% 標準 | 保留卡池落差 |
| 第 3 張 | 25% 偏強 / 50% 標準 / 25% 偏弱 | 給強卡期待 |

額外限制：

```md
同一次 3 張卡不得同名。
同一次 3 張卡盡量不要全是同類型。
若三張都是同類型，重抽第 3 張。
```

### 機率檢查

粗模擬 100,000 次：

| 出卡方式 | 三張都偏弱 | 至少一張強卡 | 至少一張標準 | 兩弱且沒強卡 |
|---|---:|---:|---:|---:|
| 純權重 35/45/20 | 約 4.3% | 約 48.7% | 約 83.5% | 約 21.0% |
| 純權重 + 三弱保底 | 0% | 約 48.8% | 約 87.7% | 約 20.9% |
| 固定槽位 | 0% | 約 25.1% | 100% | 約 14.4% |

判斷：

```md
固定槽位比較適合 MVP。
它保證每次至少有一張標準卡，也讓強卡不是每次都出。
```

### 類型保護

卡片類型分為：

```md
資源型
能力型
核心型
流派型
```

初版類型保護：

```md
同一次 3 張卡，如果三張同類型，重抽第 3 張。
偏強卡同時最多出 1 張。
```

目的：

- 避免三張都是資源卡，玩家覺得沒有戰鬥選擇。
- 避免三張都是核心卡，隊伍只是在比數值。
- 避免多人投票時完全沒有討論空間。

## 第 20 / 30 關出卡規則

第 20 / 30 關先沿用同一套槽位規則，但使用更高價值點。

```md
第 10 關：100 價值點
第 20 關：150 價值點
第 30 關：200 價值點
```

第 20 關注意事項：

```md
第 20 關卡片會影響 21-30 正式曲線。
目前 21-30 是 MVP 阻擋區，因此第 20 關卡片先只做規格，不急著細測通關公平性。
```

第 30 關注意事項：

```md
第 30 關後進 Endless。
第 30 關卡片可以更誇張，但要等 Endless 曲線確定後再細測。
```

## 卡片顯示格式

### 目前問題

多人投票時，玩家不能只看到一串數值。

玩家需要快速理解：

- 這張卡是資源、輸出、防守，還是風險玩法。
- 這張卡適合現在的隊伍狀態嗎。
- 這張卡是不是會改變玩家行為。
- 這張卡有沒有代價。

### MVP 顯示格式

每張卡顯示：

```md
卡名
類型標籤
一句效果
流派標籤
風險標籤（若有）
```

範例：

```md
金輪核心
[核心] [攻速流]
核心攻速 +0.20
```

```md
貪礦契約
[流派] [偷挖流] [有代價]
夜晚挖礦效率 +20%，核心血量上限 -10
```

### 類型標籤

| 類型 | 顯示標籤 | 用途 |
|---|---|---|
| 資源型 | [資源] | 幫隊伍蓋更快、挖更多 |
| 能力型 | [能力] | 強化玩家行動效率 |
| 核心型 | [核心] | 直接強化核心塔 |
| 流派型 | [流派] | 改變打法或帶有代價 |

### 流派標籤

| 流派 | 代表卡 |
|---|---|
| 建築成長 | 老礦工手感、土倉補給、背簍加固 |
| 高價資源 | 右礦通行證 |
| 攻擊流 | 鐵牙核心 |
| 攻速流 | 金輪核心 |
| 血量流 | 厚土外殼 |
| 修復流 | 縫補本能、夜修班 |
| 射程流 | 沙眼備料、沙眼瞭望 |
| 群怪流 | 鑽光折射 |
| 偷挖流 | 貪礦契約 |
| 高塔 / 延伸流 | 高塔工法 |

### 風險標籤

帶負面效果或會鼓勵危險玩法的卡，要加風險標籤。

| 卡片 | 風險標籤 | 原因 |
|---|---|---|
| 夜修班 | [有代價] | 夜晚挖礦效率下降 |
| 貪礦契約 | [有代價] [危險] | 鼓勵夜晚離家，且核心血量下降 |
| 高塔工法 | [有代價] | 修復效率下降 |

## 多人投票理解規則

### 目前問題

多人投票時，如果卡片只顯示數字，隊伍很難討論。

例如：

```md
核心攻速 +0.20
背負 +25
夜晚挖礦 +20%，核心血量 -10
```

新手可能不知道哪張適合現在局勢。

### 建議顯示

投票畫面中，每張卡顯示一個短定位：

| 定位 | 說明 |
|---|---|
| 穩定 | 適合大多數情況，不太會害隊伍 |
| 防守 | 適合核心常掉血、修復壓力大 |
| 資源 | 適合缺方塊、想擴張建築 |
| 輸出 | 適合怪清太慢、容易進加時 |
| 風險 | 效果強，但可能讓隊伍更危險 |

範例：

```md
鐵牙核心
[核心] [攻擊流] [輸出]
核心攻擊 +2
```

```md
厚土外殼
[核心] [血量流] [防守]
核心血量上限 +25，並回復 25
```

```md
貪礦契約
[流派] [偷挖流] [風險]
夜晚挖礦效率 +20%，核心血量上限 -10
```

### MVP 不做的事

MVP 先不做：

```md
複雜卡片評分。
AI 推薦選哪張。
根據隊伍狀態動態改卡片文字。
```

先只做固定標籤，讓玩家能快速討論。

## 卡片稀有度暫定

MVP 不做正式稀有度，只用強度層作為出卡權重。

| 強度層 | 對玩家顯示 | 內部用途 |
|---|---|---|
| 偏弱 / 功能卡 | 不顯示稀有度 | 保留卡池落差 |
| 標準卡 | 不顯示稀有度 | 主要選項 |
| 偏強卡 | 不顯示稀有度 | 低頻期待 |

原因：

- 現在卡池只有 18 張，顯示稀有度會太早把玩家引導成只選稀有。
- MVP 先驗證選擇是否有趣，不需要完整抽卡稀有度系統。
- 之後如果卡池擴到 30 張以上，再考慮稀有度。

## MVP 收尾

Must Solve 5 的 MVP 版已完成：

```md
18 張第 10 關卡池已建立。
100 / 150 / 200 價值點規則已建立。
第 20 / 30 關可用同模板 1.5x / 2x 升階。
18 張卡片已完成粗測，沒有卡片直接破第 20 關。
偏弱 / 標準 / 偏強 / 需監控分層已建立。
固定 3 槽位出卡規則已建立。
卡片顯示標籤與多人投票理解規則已建立。
```

後續保留：

```md
第 20 / 30 關高階卡等 21-30 正式曲線後再細測。
鑽光折射、沙眼瞭望、高塔工法等卡需等格子化與怪物比例變動後重測。
卡池擴到 30 張以上時，再討論正式稀有度。
```
