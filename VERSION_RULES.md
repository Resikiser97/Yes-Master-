# VERSION_RULES.md — 版本號規則

> 類型：**文件優先**（人工維護）。
> 用途：定義版本號格式、誰能動哪一段、進位邏輯。AI 必須嚴格遵守。

---

## 1. 版本號格式

```
v0.x.y.z
```

| 段 | 名稱 | 誰能改 | 何時進位 |
|---|---|---|---|
| `v0` | 開發階段 | **只有開發者** | 正式發布才改 |
| `x` | 賽季版本 | **只有開發者** | 升級後排行榜 / 存檔重置 |
| `y` | 功能版本 | **AI 可進位** | 新功能 / 重構（同時把 z 歸零） |
| `z` | 修復版本 | **AI 可進位** | bug fix |

> **AI 永遠不准動 `v0` 和 `x`。** 只能動 `y` 和 `z`。

---

## 2. 進位規則

```
新功能 / 重構：y + 1，z 歸零
  例：v0.1.3.5 → v0.1.4.0

修復 bug：z + 1
  例：v0.1.3.5 → v0.1.3.6
```

---

## 3. 易錯提醒

```
⚠️ y、z 是「獨立數字」，不是小數，會超過 9：
   v0.1.9.0 做 y+1 → v0.1.10.0   （正確）
   v0.1.9.0 做 y+1 → v0.2.0.0     （❌ 錯，那是動到 x）

⚠️ 進 y 時 z 一定歸零；進 z 時 y 不變。
```

---

## 4. 混合情況

```
同一次同時有「新功能」與「修復」→ 取最高等級（新功能優先）。
  例：本次既加了功能又修了 bug → 視為功能版本，y+1、z 歸零。
```
